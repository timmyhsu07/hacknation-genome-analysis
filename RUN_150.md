# Runbook — train the real pipeline on a 150-genome E. coli cohort

Everything needed to reproduce the real-data run on a **fresh laptop**. The heavy,
machine-specific artifacts are intentionally NOT in git (see `.gitignore`): the
`.mamba` conda env (~2 GB), the AMRFinderPlus database (~300 MB), and downloaded
genome FASTAs (~750 MB for 150). You install/fetch those locally with the steps
below. Only source, scripts, and a small worked example (`data/bvbrc_ecoli/`,
a 40-genome real run) travel in the repo.

Defensive scope: every step only detects and tabulates resistance that already
exists. Nothing here designs, mutates, or synthesizes a sequence.

## 0. Disk budget (why this laptop ran out)

| Item | Approx size | In git? |
|---|---|---|
| AMRFinderPlus conda env (`.mamba/`) | ~2 GB | no — install locally |
| AMRFinderPlus database (`amrdb/`) | ~300 MB | no — download locally |
| 150 genome FASTAs (`data/bvbrc_150/genomes/`) | ~750 MB | no — fetch locally |
| Module 1/2 outputs for 150 | ~50 MB | no |
| Repo source + scripts + 40-genome example | a few MB | yes |

Make sure the target laptop has **~4 GB free** before starting.

## 1. Clone and build the three venvs

```bash
git clone https://github.com/timmyhsu07/MAGI-Microbial-Analysis-for-Genomic-Inhibitors.git
cd MAGI-Microbial-Analysis-for-Genomic-Inhibitors
for m in module1_genome_reader module2_predictor; do
  (cd "$m" && uv venv --python 3.11 .venv && uv pip install --python .venv/bin/python -e '.[test]')
done
(cd module3_decision_report \
  && uv venv --python 3.11 .venv \
  && uv pip install --python .venv/bin/python -e '.[app,test]')
# wire Module 2 into Module 3's venv for the real decision pipeline:
uv pip install --python module3_decision_report/.venv/bin/python -e module2_predictor
```

## 2. Install AMRFinderPlus + database (the big external dependency)

```bash
# Installs ncbi-amrfinderplus into a conda/mamba env and downloads the DB.
module1_genome_reader/scripts/setup_amrfinder.sh
#   AMRFINDER_ENV=amrfinder  AMRFINDER_DB_DIR=module1_genome_reader/amrdb
```
Verify: `<amrfinder-bin> -d module1_genome_reader/amrdb/latest -l` lists `Escherichia`.
Note the resolved amrfinder binary path — call it `$AMR` below.

## 3. Fetch the 150-genome labelled cohort from BV-BRC

```bash
python3 scripts/fetch_bvbrc_ecoli.py \
  --max-genomes 150 --limit-per-drug 20000 \
  --out-dir data/bvbrc_150
```
Check the printed per-drug class balance. If **gentamicin R** is thin, raise
`--max-genomes` (resistance to it is genuinely rarer). Output:
`data/bvbrc_150/genomes/*.fna`, `labels.csv`, `target_genes.csv`.
(The script uses certifi for TLS, so it works even where system Python has no CA store.)

## 4. Annotate with Module 1 (the long step: ~1–3 min/genome)

Simple path — native parallelism:
```bash
PATH="<conda-env-bin>:$PATH" module1_genome_reader/.venv/bin/genome-reader \
  --input-dir data/bvbrc_150/genomes \
  --output-dir data/bvbrc_150/m1_out \
  --organism Escherichia \
  --database-dir module1_genome_reader/amrdb/latest \
  --amrfinder-bin "$AMR" \
  --cache-dir data/bvbrc_150/cache --workers 8
```
Annotations are cached by genome checksum, so re-running resumes cheaply. For the
fan-out (30 cmux workers × 5 genomes) variant, `scripts/shard_150.sh` splits the
cohort into `data/bvbrc_150/batches/`; run each batch against the shared
`--cache-dir`, then run the command above once to assemble a single consistent
matrix from cache.

Verify: `data/bvbrc_150/m1_out/features_binary.parquet` has ~150 rows.

## 5. Train Module 2 (real cross-validation)

```bash
cat > data/bvbrc_150/train_config.yaml <<'YAML'
seed: 1234
output_dir: m2_out
inputs:
  feature_matrix: m1_out/features_binary.parquet
  feature_schema: m1_out/feature_schema.json
  labels: labels.csv
  target_gene_table: target_genes.csv
distance: {method: jaccard, mash_sketch_dir: null, cluster_threshold: 0.05}
cv: {n_splits: 5, calibration_fraction: 0.25, min_per_class: 3}
model: {C: 1.0, max_iter: 2000}
ood: {threshold: 0.5}
decision: {min_confidence: 0.65}
drugs:
  ciprofloxacin: {target_genes: [gyrA, parC], on_target_absent: no_call}
  ampicillin:    {target_genes: [ftsI], on_target_absent: no_call}
  gentamicin:    {target_genes: [rpsL], on_target_absent: no_call}
YAML

(cd data/bvbrc_150 && ../../module2_predictor/.venv/bin/python -m predictor.cli --config train_config.yaml)
```
`data/bvbrc_150/m2_out/metrics_report.json` now carries, per drug and per genetic
group: balanced accuracy, separate resistant/susceptible recall, AUROC, PR-AUC,
Brier, **no_call_rate + accuracy_on_called**, plus reliability PNGs.

## 6. Decision cards through Module 3

```bash
sed 's#bvbrc_smoke#bvbrc_150#' data/bvbrc_smoke/run_m3.py > data/bvbrc_150/run_m3.py 2>/dev/null \
  || cp data/bvbrc_ecoli/run_m3.py data/bvbrc_150/run_m3.py
module3_decision_report/.venv/bin/python data/bvbrc_150/run_m3.py
```
Or launch the UI (real mode auto-fills paths from `MAGI_*` environment variables):
```bash
MAGI_MODELS_DIR=data/bvbrc_150/m2_out/models \
MAGI_TARGET_GENES=data/bvbrc_150/target_genes.csv \
MAGI_MODULE1_OUT=data/bvbrc_150/m1_out \
MAGI_SPECIES="Escherichia coli" \
module3_decision_report/.venv/bin/python -m streamlit run module3_decision_report/src/decision_report/app.py
```

## Sanity check

`data/bvbrc_ecoli/` in the repo is a completed **40-genome** real run — compare
your 150-genome `metrics_report.json` against it to confirm the wiring matches.
